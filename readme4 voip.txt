
فرآیند کار به این شکله که سرویس ویپ شما، وقتی یک بیمار نوبت رو رزرو می‌کنه، یک درخواست HTTP به آدرسی که شما تعیین می‌کنید (همون Webhook) می‌فرسته و این درخواست، حاوی اطلاعات تماس و نوبت هستش .

مراحل گام‌به‌گام اتصال
۱. آدرس Webhook (همون API که ساختیم) رو به سرویس ویپ بدید

آدرسی که قبلاً ساختیم و تست کردیم رو باید به عنوان آدرس Webhook در پنل مدیریت سرویس ویپ خودتون ثبت کنید:

text
http://your-domain.com/api/webhook/appointment
این آدرس رو به تیم فنی یا پشتیبانی سرویس ویپ (مثل شرکت "همکاران" یا هر ارائه‌دهنده دیگه‌ای) بدید .

۲. فرمت داده‌هایی که سرویس ویپ باید ارسال کنه رو مشخص کنید

سرویس ویپ باید داده‌های نوبت رو به صورت JSON به آدرس بالا ارسال کنه . ما قبلاً این ساختار رو تعریف کردیم و تست کردیم. اون رو به تیم فنی سرویس ویپ بدید:

json
{
    "patient_national_code": "1234567890",
    "patient_name": "رضا کریمی",
    "patient_mobile": "09123456787",
    "patient_email": "reza@example.com",
    "doctor_name": "دکتر علی محمدی",
    "appointment_date": "2026-07-01",
    "appointment_time": "10:00",
    "appointment_type": "in_person",
    "notes": "ویزیت از طریق ویپ کلینیک"
}
۳. نحوه عملکرد در سرویس ویپ

معمولاً در پنل مدیریت سرویس ویپ، یک بخش به نام Webhook یا Integration وجود داره . توی اون بخش:

آدرس Webhook رو وارد می‌کنید.

مشخص می‌کنید که چه رویدادهایی (مثل تماس جدید یا ثبت نوبت) باعث ارسال Webhook بشن .

قالب داده‌های ارسالی رو طبق فرمت بالا تنظیم می‌کنید.

سناریوی واقعی: اتصال به سرویس ویپ "همکاران"
برای مثال، اگر از سرویس تلفن ابری همکاران استفاده می‌کنید، Webhook اون‌ها رویدادهایی مثل Newstate (وضعیت تماس)، Hangup (قطع تماس) و Cdr (اطلاعات تماس) رو ارسال می‌کنه . اما برای ثبت نوبت، سرویس ویپ باید یک رویداد سفارشی (Custom Event) تعریف کنه که دقیقاً همون داده‌هایی که ما نیاز داریم رو به سیستم شما بفرسته. شما باید با تیم فنی اون سرویس هماهنگ کنید که این رویداد رو به آدرس Webhook شما ارسال کنن.

تنظیمات سمت سیستم خودتون (تکمیل کار)
۱. Webhook رو در سیستم خودتون فعال کنید

قبل از اینکه سرویس ویپ شروع به ارسال داده کنه، مطمئن بشید که Webhook توی سیستم خودتون فعال هستش.

bash
TOKEN="11|3fPLnO3lewQBR6ylxWXG8VL9qzYYwmAlaWqYqRga632fa4f7"

curl -X POST http://localhost:8210/api/admin/webhook/toggle \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"enabled": true}'
۲. غیرفعال کردن Webhook (در صورت نیاز)

اگر خواستید هر زمانی Webhook رو متوقف کنید:

bash
curl -X POST http://localhost:8210/api/admin/webhook/toggle \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"enabled": false}'
۳. بررسی لاگ‌ها و رفع خطا

بعد از اینکه سرویس ویپ تنظیم شد و اولین درخواست‌ها رو ارسال کرد، می‌تونید لاگ‌ها رو چک کنید تا مطمئن بشید همه چی درست کار می‌کنه یا خطاها رو پیدا و رفع کنید.

bash
curl -X GET http://localhost:8210/api/admin/webhook/logs \
  -H "Authorization: Bearer $TOKEN"
جمع‌بندی نهایی
آدرس Webhook رو به سرویس ویپ بدید: http://your-domain.com/api/webhook/appointment

فرمت JSON رو برای ارسال داده بهشون بدید.

توی پنل سرویس ویپ Webhook رو فعال کنید.

توی سیستم خودتون Webhook رو فعال کنید (enabled: true).

لاگ‌ها رو چک کنید تا مطمئن بشید درخواست‌ها به درستی دریافت و پردازش میشن.

با این کار، هر بار که یک بیمار از طریق سیستم ویپ نوبت می‌گیره، به‌طور خودکار توی سیستم کلینیک شما هم ثبت میشه.
